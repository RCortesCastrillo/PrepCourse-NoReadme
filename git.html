para crear nuestro repositorio local para eso utilizamos el comando (git init) esto le dice a git "git empieza a guardar todos los archivos y carpetas dentro de esta carpeta en la que estoy " todos los comandos de git iniciaran con la palabra clave (git ) de esta forma podemos diferenciar cuando un comando es de git y cuando no al realizar el comando git init automaticamente la carpeta en la que nos encontrabamos posicionados desde la terminal se convirtio en nuestro nuevo repositorio local osea el almacen en donde vamos a guardar todos nuestros archivos de codigo e inclusive otras carpetas podemos corroborar esto en el mensaje que nos dio la terminal (inicialized empty repository) antes la direccion que marcaba la terminal terminaba en el nombre de nuestra carpeta pero ahora termina con (master) esto nos ayudara a reconocer que esta carpeta es un nuevo repositorio local ls -a (la opcion "a" nos permite ver archivos y carpetas ocultos ) al realizar este comando podemos ver que ahora dentro de nuestra ubicacion se encuentra una carpeta llamada (.git) esta carpeta (.git) es la que se encargara de llevar a cabo todo el seguimiento de GIT y nunca debe ser borrado --> como podemos ver nos dice no commits yet eso quiere decir que no hemos guardado nada la palabra commits(significa guardar )y cada ves que guardemos habran mas commits Ahora nuestro repositorio local fue creado los archivos y carpetas que se encuentren ahi ya sean nuevos o existenesnestan en una de las fases o estados de seguimiento que tiene git que es un commit? captura instantanea de los cambios preparados en ese momento del proyecto git no las cambiara nunca a menos que nosotros lo pidamos expresamente resumen guarda una foto del estado de los archivos y carpetas del repositorio local, al momento de ser tomada
